# Social Commerce APP Experiment - (LoopSell) (V1)

**Status**: Learning Prototype | **Stage**: Early Development | **Purpose**: Prototyping/Learning

An experimental social commerce app exploring:
- Ephemeral content (24-hour posts with auto-archive)
- Shopping cart integration
- Real-time media/text status updates
- Real-time messaging and media sharing
- Interaction engagement (Likes, Shares, Save, Views, Comments)

Tech Stack: Expo, React Native, Firebase, NodeJS

Implemented Features
- 24-Hour Posts: Auto-archiving feed content
- Shopping Cart: Add/remove items to cart that forwards to inbox
- Status Updates: Real-time user status feed
- Bottom Sheet Archive: Expired posts organized view
- Like, Share, Save, Comment: On each item post

What I've mastered from this project:
- Firebase Firestore real-time listeners
- React Native navigation patterns
- Expo managed workflow
- Social media-style feed implementation

Known Issues (being addressed in v2):
- UI inconsistencies
- Slow load times
- Memory leaks 
- Cart sync delays

What I'm improving:
- Performance optimization 
- UI polish
- State management refactor
- Skeleton loading states
- Optimized Firebase queries
